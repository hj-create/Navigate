Navigate is designed with accessibility in mind. Our platform is usable by people with visual, motor, cognitive, and hearing disabilities. Our Commitment to Accessibility

Navigate is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and apply the relevant accessibility standards to ensure our website is usable by all, including those with disabilities.

WCAG 2.1 Compliance

Navigate conforms to WCAG 2.1 Level AAA, which represents the highest level of accessibility. This includes:

  • Level A: Foundational accessibility requirements ‚úì
  • Level AA: Enhanced accessibility for broader audiences ‚úì
  • Level AAA: Advanced accessibility for specialized needs ‚úì

Accessibility by Disability Type

Visual Disabilities

  • Descriptive alt text for all images and graphics
  • WCAG AAA compliant color contrast ratios
  • Text remains readable when magnified up to 200%
  • No reliance on color alone to convey information
  • Support for screen magnifier software

Motor & Mobility Disabilities

  • Full keyboard navigation without mouse required
  • Visible focus indicators on all interactive elements
  • Touch-friendly button and link sizes (minimum 44x44 pixels)
  • No time limits on form submission
  • Skip-to-content links for faster navigation

Cognitive & Learning Disabilities

  • Clear, simple language throughout the site
  • Consistent navigation and layout patterns
  • Dyslexia-friendly sans-serif font
  • Proper use of headings and lists for structure
  • Descriptive form labels and error messages

Hearing Disabilities

  • All information conveyed through text and visuals (no audio-only content)
  • Text-based chat and messaging systems
  • Video captions available for all video content
  • No reliance on sound for important information

Assistive Technology Support

Navigate is compatible with the following assistive technologies:

  • Screen readers (JAWS, NVDA, VoiceOver, TalkBack)
  • Screen magnifiers
  • Voice control software (Dragon, Windows Voice Control)
  • Alternative keyboards and input devices
  • Switch control devices
  • Browser zoom and high contrast settings

Known Limitations

We strive for complete accessibility, but some limitations may exist:

  • Third-party embedded content (YouTube videos) may have varying accessibility support
  • Some older browser versions may not support all accessibility features
  • PDF files generated by users may not be fully accessible

Citations & References

The accessibility guidelines and standards referenced in this statement are based on the following authoritative sources:

  1. Web Content Accessibility Guidelines (WCAG) 2.1
    W3C (World Wide Web Consortium)
    https://www.w3.org/WAI/WCAG21/quickref/
    Official reference for WCAG 2.1 Level A, AA, and AAA compliance criteria.
  2. Web Accessibility Initiative (WAI)
    W3C (World Wide Web Consortium)
    https://www.w3.org/WAI/
    Comprehensive resource for web accessibility guidelines and best practices.
  3. Americans with Disabilities Act (ADA) - Section 508
    U.S. Access Board
    https://www.access-board.gov/
    U.S. federal accessibility standards for electronic and information technology.
  4. ARIA Authoring Practices Guide (APG)
    W3C (World Wide Web Consortium)
    https://www.w3.org/WAI/ARIA/apg/
    Best practices for using Accessible Rich Internet Applications (ARIA) attributes.
  5. Color Contrast Standards (WCAG AAA)
    W3C - WCAG 2.1 Criterion 1.4.6
    https://www.w3.org/WAI/WCAG21/Understanding/contrast-enhanced.html
    Minimum color contrast ratio requirement: 7:1 for AAA compliance.
  6. Touch Target Size Guidelines
    Apple Human Interface Guidelines & WCAG 2.1 Criterion 2.5.5
    https://www.w3.org/WAI/WCAG21/Understanding/target-size.html
    Minimum touch target size: 44x44 pixels (Apple) or 24x24 CSS pixels (WCAG).
  7. Keyboard Navigation Standards
    W3C - WCAG 2.1 Criterion 2.1.1
    https://www.w3.org/WAI/WCAG21/Understanding/keyboard.html
    All functionality must be accessible via keyboard without requiring specific timing.
  8. Screen Reader Compatibility
    W3C - WCAG 2.1 Criterion 1.3.1 (Info and Relationships)
    https://www.w3.org/WAI/WCAG21/Understanding/info-and-relationships.html
    Semantic HTML and ARIA attributes must properly convey structure and relationships.
  9. Motion & Animation Guidelines
    W3C - WCAG 2.1 Criterion 2.3.3 (Animation from Interactions)
    https://www.w3.org/WAI/WCAG21/Understanding/animation-from-interactions.html
    Respect user's prefers-reduced-motion preferences to prevent vestibular disruption.
  10. Text Readability & Zoom Support
    W3C - WCAG 2.1 Criterion 1.4.4 (Resize Text)
    https://www.w3.org/WAI/WCAG21/Understanding/resize-text.html
    Text must be resizable to 200% without loss of content or functionality.
  11. Colorblind-Friendly Design
    W3C - WCAG 2.1 Criterion 1.4.1 (Use of Color)
    https://www.w3.org/WAI/WCAG21/Understanding/use-of-color.html
    Color must not be the only visual means of conveying information.
